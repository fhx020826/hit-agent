# ClaudeCode 启动 Prompt

```text
你现在是“前沿融课教师助手”项目的主开发代理。请不要先泛泛讨论方案，而是读取我提供的全部背景文件后，直接进入 MVP 开发。

【你将同时收到的文件】
1. 赛事说明.txt
2. 哈工大智能体平台校内建设者标准操作过程参考 v2.docx
3. 教师赛道-前沿融课教师助手.xlsx
4. 00_交付清单与使用说明.docx
5. 01_项目背景与赛事约束说明.docx
6. 02_产品功能说明书_面向用户.docx
7. 03_技术方案说明书_面向开发.docx
8. 04_ClaudeCode_开发任务书.docx

【项目主线】
本项目不是通用聊天机器人，也不是普通 PPT/教案生成器，而是一个面向高校教师的“前沿知识快速入课”智能体。核心闭环必须是：
教师创建课程画像 -> 上传课程与前沿资料 -> 系统生成结构化课时包 -> 教师发布给学生端 -> 学生围绕当前课时包提问 -> 系统汇总学生问题并生成教师复盘建议。

【你必须遵守的原则】
1. 第一优先级是“可运行、可演示、闭环完整”。
2. 先做比赛 MVP，不做大全套平台。
3. 所有 AI 输出优先结构化 JSON。
4. 所有问答与生成尽量基于 RAG 或可替换的 mock RAG。
5. 如果真实模型、真实检索、真实平台接口暂时未打通，允许你先用 mock/demo 数据跑通链路，但必须保留后续替换空间。
6. 不要把项目做偏成泛用型智能助手。
7. 不要省略教师复盘页，这部分是产品差异化关键。
8. 如果遇到实现分歧，优先选择更稳、更快、更适合比赛演示的方案。

【必须实现的 MVP】
教师端：
- 创建课程画像
- 上传课程资料与前沿资料
- 触发前沿融课分析
- 生成结构化课时包
- 查看/编辑/保存/发布课时包
- 查看教师复盘页

学生端：
- 选择课程/章节或已发布课时包
- 围绕当前课时包提问
- 获得课程边界内的回答
- 查看延伸阅读或参考依据（可简化）

分析端：
- 汇总学生问答日志
- 输出高频问题
- 输出易混淆概念
- 输出知识盲区
- 输出教学建议

【推荐开发顺序】
阶段 1：
- 先给出技术栈建议
- 给出仓库目录结构
- 初始化项目骨架
- 搭建最小前后端路由与页面

阶段 2：
- 完成课程创建页
- 完成课时包结果页
- 先用 mock 数据跑通：课程创建 -> 课时包生成 -> 展示结果

阶段 3：
- 增加资料上传
- 增加最小文档解析/文本存储
- 增加基础检索或 mock 检索

阶段 4：
- 完成学生问答页
- 记录问答日志
- 加入边界约束

阶段 5：
- 完成教师复盘页
- 生成高频问题、混淆点、建议
- 准备 demo seed 数据和演示链路

【你的输出格式要求】
每一阶段结束后都必须明确输出：
1. 已完成内容
2. 修改/新增的文件
3. 当前如何运行
4. 当前页面入口
5. 下一步计划

【现在请直接开始】
请按以下顺序输出并执行：
1. 你建议的技术栈
2. 仓库目录结构
3. 第一阶段具体实施计划
4. 然后直接开始编写 MVP 代码，不要停留在讨论层面。

```
