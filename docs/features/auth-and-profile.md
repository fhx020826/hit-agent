# 统一认证与个人中心

## 1. 目标
本模块统一处理登录、注册、角色识别、个人资料维护与设置入口收纳，解决教师和学生入口分散、页面重复的问题。

## 2. 当前实现
- 页面右上角统一展示登录 / 注册入口
- 注册与登录共用一个弹窗，先选角色，再进入对应表单
- 登录成功后返回账号角色、资料信息和 Bearer Token
- 前端根据角色自动展示教师或学生功能视图
- 登录后右上角显示个人头像，点击后打开顶部固定浮层菜单
- 个人信息维护页面为 `/profile`
- 设置中心页面为 `/settings`

## 2.1 角色说明
- `admin`：全站管理员，可管理用户、课程、讨论空间与平台数据
- `teacher`：教师，只能管理与自己课程相关的教学数据
- `student`：学生，只能查看自己的资料与所属课程空间

## 3. 注册规则
### 教师注册字段
- 姓名
- 工号
- 学院
- 专业方向
- 所属教研室
- 教学组
- 岗位称谓
- 邮箱
- 手机号
- 密码

### 学生注册字段
- 姓名
- 学号
- 学院
- 专业
- 年级
- 班级
- 邮箱
- 手机号
- 密码

## 4. 登录与账号约束
- 每个学生必须使用本人账号登录
- 账号体系下的提问、作业、反馈、学习记录与本人绑定
- 匿名发言不等于匿名登录
- 匿名发言只是在教师端和其他展示层隐藏公开身份

## 5. 设置中心结构
- 个人信息：查看和修改角色相关资料
- 外观设置：主题模式、主色、字体、皮肤、语言
- 账号信息：当前账号、当前角色、退出登录
- 账号安全：修改密码
- 头像管理：支持默认头像或上传自定义头像

## 6. 右上角入口行为
- 未登录时显示 `登录` / `注册`
- 已登录时显示用户头像
- 点击头像后，菜单固定漂浮在页面顶部右侧，不跟随内容流下沉
- 菜单中可进入个人中心、设置中心、修改密码和退出登录
## 7. 当前边界
- 当前未接入短信验证码或第三方单点登录
- 管理员账号不支持前台自助注册，只能由管理员后台创建或使用系统演示账号登录
