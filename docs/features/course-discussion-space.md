# 课程讨论空间

## 1. 定位
课程讨论空间以“课程 + 授课班级”为单位自动创建，不是全站公共论坛。它用于把学生、教师、课程专属 AI 助教放进同一个学习协作空间，形成围绕课程内容的共同学习场景。

## 2. 创建逻辑
- 教师创建课程并填写授课班级后，系统自动创建对应讨论空间
- 任课教师自动加入空间
- 同班学生根据 `user_profiles.class_name` 自动加入
- 空间中默认存在一个 `AI 助教` 成员

## 3. 空间成员与消息类型
- 成员：教师、学生、AI 助教
- 支持消息：文本、图片、文档、多附件消息
- 支持匿名：学生可按条切换实名 / 匿名发送
- 支持 `@AI 助教`：被触发时才参与回答，避免刷屏

## 4. AI 群聊回答逻辑
当学生 `@AI 助教` 或勾选“请 AI 回答”时：
1. 读取最近一段群聊消息
2. 读取当前课程资料与课程画像
3. 读取当前消息上传的可解析附件
4. 组织成更贴近群聊语境的 Prompt
5. 调用真实大模型生成回答

AI 会在回答来源中体现：
- 最近讨论上下文
- 课程资料
- 上传文件分析
- 大模型补充解释

## 5. 匿名机制
- 匿名仅是展示层匿名
- 匿名消息统一显示为 `匿名同学`
- 普通用户不能通过成员搜索或发言筛选反推出匿名消息真实发送者
- 教师查看“某个实名成员发言”时，只返回该成员实名发送的消息

## 6. 聊天记录查询
- 支持关键词搜索
- 支持按实名成员名字搜索公开发言
- 支持按 `teacher / student / ai` 筛选
- 支持定位消息上下文
- 查询范围严格限制在当前讨论空间内

## 7. 文件规则
- 讨论空间消息可以绑定附件
- 空间成员可下载该空间内消息附件
- 非空间成员不能访问附件文件

## 8. 当前边界
- 当前版本主要支持基于接口轮询与页面刷新后的消息同步，未对普通群聊消息做复杂已读状态管理
- AI 只在被触发时回答，不会自动总结全空间所有问题
