# UI 与交互规范

## 1. 栏目选中态规范
本项目已统一取消“纯黑底选中”方案，改为柔和浅色高亮，避免文字与背景对比不清。

### 统一原则
- 选中项不再使用纯黑或过深背景色
- 选中态优先使用与主题主色一致的浅色高亮面
- 同时配合更深文字、边框、轻微阴影和高亮条强化当前定位
- 教师端与学生端保持一致的交互语言

### 当前通用样式类
- `ui-pill`
- `ui-pill-active`
- `ui-card-active`
- `ui-tab-active`
- `sticky-toolbar`

## 2. 覆盖范围
当前已统一覆盖以下位置：
- 顶部导航栏
- 角色页常用栏目切换
- 问答页历史会话切换
- 作业页筛选与选中项
- 教师端问题列表选中态
- 登录 / 注册弹窗中的模式与角色切换
- 设置页与外观切换卡片
- 模型选择浮层中的当前选中模型

## 3. 多主题兼容要求
### 白天模式
- 选中态采用浅主色背景 + 深色文字
- 保持对比清楚但不过分刺眼

### 夜间模式
- 选中态避免深灰叠深灰
- 使用亮度更高但不过曝的强调面

### 护眼模式
- 选中态保持低饱和、柔和边界
- 避免高纯度蓝紫色造成视觉刺激

## 4. 模型选择器规范
### 布局原则
- 模型选择器不再放在内容区底部
- 相关页面优先采用顶部工具条或顶部悬浮栏
- 下拉浮层需使用足够高的 `z-index`
- 当前所选模型必须始终可见

### 当前实现
#### 学生问答页
- 采用顶部悬浮模型选择器
- 列表在浮层中展开，可滚动查看
- 每个模型展示名称、用途说明、实际模型名和接入状态

#### 教师 PPT / 教案更新页
- 同样采用顶部悬浮模型选择器
- 生成前即可确认当前实际使用模型
- 历史记录中会保留当次使用模型与调用状态

## 5. 错误展示原则
- 模型不可用时，不允许静默假装回答成功
- 前端需要明确展示“当前模型服务暂不可用”“请先配置模型”等信息
- 回答来源区或生成结果区应显示实际模型与调用状态，便于演示和排查
