# 课程问答与多轮记忆

## 1. 目标
把学生问答从“只会引用课时包”升级为“课程资料检索 + 大模型生成 + 多轮连续对话 + 教师协同反馈”的综合系统，并确保模型选择真实生效。

## 2. 问题分发机制
### 仅 AI 回答
- 系统优先检索课程资料、课程包和可解析附件
- 当课程资料不足时自动调用当前所选模型补充解释
- 前端会把 `selected_model` 传给后端，后端按模型路由真实调用

### 仅教师回答
- 问题直接进入教师待处理列表
- 学生端显示“教师待回复”状态
- 不自动生成正式 AI 回答

### AI + 教师
- AI 先即时回答
- 同时为教师生成站内提醒
- 教师可后续补充、纠正或延伸说明

## 3. 实际调用链路
1. 学生输入问题文字，并可附带图片、文档或压缩包
2. 前端读取当前选中的模型 `selected_model`
3. 后端先整理课程资料、可解析附件内容和会话历史
4. 系统将检索结果、上下文记忆、附件摘要与用户问题一起组织为 Prompt
5. 后端根据模型标识调用真实模型服务
6. 返回真实生成内容
7. 前端展示答案，并在来源区显示本次实际使用模型、提供方、调用状态和耗时
8. 问题记录中保存本次实际使用的模型键值

## 4. 多轮记忆
- 每次提问属于某个会话 `chat_sessions`
- 会话内历史问题会一并传给模型作为上下文
- 学生可在历史会话中继续追问

## 4.1 回答格式化展示
- 前端会对模型返回内容做换行标准化处理
- 能识别 `\\n`、`/n` 等错误换行写法，并转换为真实段落
- 连续的分点内容会按列表方式渲染，避免整段文字堆在一起
- 教师端查看 AI 回答时也沿用同一套格式化组件，保证展示一致

## 5. 模型选择与失败处理
### 当前页面行为
- 学生问答页采用顶部悬浮模型选择器
- 当前所选模型始终可见
- 模型较多时通过高层级浮层菜单展示，不会被页面内容遮挡

### 后端行为
- 只返回当前已接入、可调用的模型列表
- 若某个模型未配置，不会伪装成可用
- 若调用失败，会明确返回“模型服务暂时不可用”等错误说明
- 开发环境下可从回答来源中直接看到实际模型与耗时

## 6. 多模态输入
### 当前支持
- 纯文本问题
- 图片问题：jpg / jpeg / png / webp
- 文档问题：txt / md / pdf / doc / docx / ppt / pptx
- 压缩包问题：zip / rar

### 当前解析边界
- 完整 / 优先解析：txt、md、csv、json、docx、pptx
- 文件索引：zip
- 主要做留存与教师查看：pdf、doc、ppt、rar
- 图片是否真正进入视觉理解，取决于所选模型是否支持视觉能力
- 若当前无法完整解析某类文件，系统会如实提示，而不是伪装成已完全理解

## 7. 匿名发言说明
- 学生必须先登录本人账号
- `is_anonymous=true` 仅影响教师端和公开展示端
- 平台内部仍保留与账号的关联，用于权限控制和历史记录查询

## 8. 学生端展示要求
- 标明回答来源：AI 或教师
- 展示时间戳
- 展示教师回复状态：待回复 / 已回复 / 已关闭
- 展示附件数量与下载入口
- 在 AI 来源区展示“本次回答使用模型 / 提供方 / 调用状态 / 耗时”
- 可一键请求教师上传课堂讲义资料，教师端收到提醒
