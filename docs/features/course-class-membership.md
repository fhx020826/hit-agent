# 课程 / 班级 / 师生关系匹配

平台新增了持续可用的课程关系入口，不再要求用户首次进入时强制绑定，但教师和学生始终都能在系统中完成课程匹配。

## 教师端

- 入口：`/teacher/course-management`
- 支持创建课程
- 支持为课程绑定一个或多个授课班级
- 支持设置学期
- 支持查看课程下已绑定成员
- 支持刷新邀请码
- 课程与班级绑定后，会自动同步讨论空间

## 学生端

- 入口：`/student/courses`
- 支持查看“我的课程”
- 支持通过邀请码加入课程
- 支持先查看课程目录、教师和可选班级
- 绑定成功后可进入课程问答、讨论空间、作业和反馈流程

## 数据关系

- `courses`：课程主体，保存教师、学期、邀请码等基础信息
- `course_classes`：课程绑定的班级列表，保存班级级别的邀请码与讨论空间关联
- `course_members`：课程成员关系表，保存教师/学生与课程、班级的显式绑定

## 功能生效规则

以下能力改为基于显式课程成员关系判断，而不是只根据个人资料中的班级字符串推断：

- 学生课程问答
- 教师布置作业与学生接收作业
- 学生匿名反馈与教师查看分析
- 课程讨论空间成员同步
- 课程资料访问与资料请求

## 兼容说明

- 历史 `class_name` 字段仍保留，便于兼容旧页面和旧数据
- 但新的权限判断会优先使用 `course_members`
- 教师仍可继续使用原有课程设计页面，课程管理页负责长期维护课程关系
